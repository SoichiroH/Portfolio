  • NWBHS Patient Form Tracker

  • Web Application

Screenshots

  • Synopsis

    • This web application was created and deployed for our client North West Behavioral Health Service (NWBHS), a non-profit organization focused on infant mental health care in Jacksonville Florida, who requested for more efficient, useful and organized method to keep track of their patients’ personal information including government required form due dates. This was a project of UNF’s most practical two-semester sequence programming course.
      Systems Senior Project I & II >

    • NWBHS has over 300 active patients, with 20 – 30 new patients coming in every month. They are responsible for submitting 3 forms (each with different due dates) to the state for every patient.
      Before deploying our application, they were managing their patient information with a Microsoft Excel sheet, in which it was not shared over the network.

    • Our solution was to build an online web application to let every employee in NWBHS to have access and view/edit the current status of each patients in a vary simple and organized way.
      Since our client was a medical service, we have used Microsoft Azure and its security to avoid the potential HIPAA compliant issue regarding personal information.

  • Main Functionality

    • User name and password are required for every user. We created hierarchical role such as manager, provider and user based on the actual organization structure. For example, only the managers can add/delete new users into the system.

    • Information for each of the patients is stored in Microsoft Azure cloud database and reflected in a table in the application for easy access for users.

    • Lists the names of patients who are waiting to be able to start treatment. Each patient’s case and basic information will be created from this page.

    • The case list page provides a place for users to view their patients, form due dates, and other details about their cases. When a patient is clicked, a modal with multiple tabs will pop up with the patient’s information and the user will be able to update its information.

    • Being that the requirements given by the state may change, the manager role has the privilege to add, remove, and edit form due dates as it is needed.

    • A manager role has the privilege to add new users and specify the role that these users will have in the system.

    • The application provides a way for its users to color code the cases that they have, which can also be used as a parameter for searches.

    • Records the user name, time and its action done to the system.
